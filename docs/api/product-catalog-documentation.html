Product Catalog

Product CRUD with inline variations, lookup CRUD (categories, colors, units), and dropdown endpoints

Integration Guide

The product form has two sections: (1) Product details — name, category, unit; (2) Variations table — each row has color, size, rate. Categories, colors, and units can be created on the fly from dropdowns. Create/Update Product accepts an optional 'variations' array for atomic product + variation management.

POST /products

Create Product

Creates a new product with a single inline variation (color + size). categoryId, colorId, and size are required at the top level — no nested variations array.

💡
Developer Note: categoryId, colorId, and size are required. materialRate is optional (defaults to 0). unitId is not accepted on create. If a product with the same name already exists, the new variation is auto-linked to that existing product instead of creating a duplicate.
Authentication Bearer Token
Permissions ADMIN, OPERATOR
cURL Example
curl -X POST "{{base_url}}/products" \
  -H "Authorization: Bearer {{authToken}}" \
  -H "Content-Type: application/json" \
  -d '{"materialName":"Cotton Roll","categoryId":1,"colorId":1,"size":"M","materialRate":500}'
Request Body
{
  "materialName": "Cotton Roll",
  "categoryId": 1,
  "colorId": 1,
  "size": "M",
  "materialRate": 500
}
Example Response 201
{
  "success": true,
  "data": {
    "id": 2,
    "materialName": "Cotton Roll",
    "materialRate": 500,
    "categoryId": 1,
    "variations": [
      {
        "variationId": 1,
        "colorId": 1,
        "size": "M",
        "materialRate": 500
      }
    ]
  }
}

PUT /products/:id

Update Product

Updates product fields and/or manages variations using a diff strategy. All fields are optional (partial update). Variations use variationId to distinguish: present = update, absent = insert, isActive:false = soft-delete.

💡
Developer Note: All fields optional. Variations diff: variationId present → update, variationId absent → insert new, variationId + isActive:false → soft-delete. Product fields are merged with existing record to prevent undefined parameter errors.
Authentication Bearer Token
Permissions ADMIN, OPERATOR
cURL Example
curl -X PUT "{{base_url}}/products/:id" \
  -H "Authorization: Bearer {{authToken}}" \
  -H "Content-Type: application/json" \
  -d '{"materialRate":550,"variations":[{"variationId":1,"colorId":1,"size":"M","materialRate":560},{"colorId":3,"size":"S","materialRate":480},{"variationId":2,"isActive":false}]}'
Request Body
{
  "materialRate": 550,
  "variations": [
    {
      "variationId": 1,
      "colorId": 1,
      "size": "M",
      "materialRate": 560
    },
    {
      "colorId": 3,
      "size": "S",
      "materialRate": 480
    },
    {
      "variationId": 2,
      "isActive": false
    }
  ]
}
Expected Status 200

POST /products/:id/variations

Add/Update Product Color Matrix (Standalone)

Standalone endpoint to add or update a single color/size pricing variation. Kept as a fallback for individual variation edits outside the product form.

💡
Developer Note: Pass variationId in body for updates. Omit variationId for new entries. Prefer using the inline variations array on PUT /products/:id for batch operations.
Authentication Bearer Token
Permissions ADMIN, OPERATOR
cURL Example
curl -X POST "{{base_url}}/products/:id/variations" \
  -H "Authorization: Bearer {{authToken}}" \
  -H "Content-Type: application/json" \
  -d '{"fkLuColorId":1,"materialRate":550,"size":"M"}'
Request Body
{
  "fkLuColorId": 1,
  "materialRate": 550,
  "size": "M"
}
