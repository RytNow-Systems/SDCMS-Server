========================================
DISPATCH & TERMINAL STATES — Test Data
Sprint 4 Feature B: Dispatch & Terminal States
========================================

--- TEST 1: Dispatch Parcels (Bulk — Happy Path) ---
Method: POST
URL: /api/v1/parcels/dispatch
Headers:
  Authorization: Bearer <OPERATOR_TOKEN>
  Content-Type: application/json

Pre-condition: Parcels must be in AWB_LINKED state

Body:
{
  "parcelIds": [1, 2]
}

Assertions:
  - Status: 200
  - Body: { success: true, data: { dispatched: 2, parcels: [{ status: "DISPATCHED", dispatchDate: "..." }, ...] } }
  - All parcels transitioned to DISPATCHED
  - DispatchDate stamped on each

Post-Request Verification:
  - GET /api/v1/parcels/1/timeline should show STATUS_UPDATE event with newStatus: "DISPATCHED"

--- TEST 2: Dispatch — Wrong State (Still PENDING) ---
Method: POST
URL: /api/v1/parcels/dispatch
Headers:
  Authorization: Bearer <OPERATOR_TOKEN>
  Content-Type: application/json

Pre-condition: Parcel not yet AWB_LINKED

Body:
{
  "parcelIds": [1]
}

Assertions:
  - Status: 400
  - Body: { success: false, error: "Cannot dispatch parcel PDS-...: status is 'PENDING'. Dispatch requires AWB_LINKED status." }
  - No dispatch before AWB is linked
