========================================
SCAN MODULE — Test Data
Sprint 4 Feature A: Two-Scan Operation (QR + AWB)
========================================

--- TEST 1: Scan + Link AWB (OPERATOR — Happy Path) ---
Method: POST
URL: /api/v1/parcels/scan
Headers:
  Authorization: Bearer <OPERATOR_TOKEN>
  Content-Type: application/json

Pre-condition: Parcel with parcelId "PDS-A1B2C3" must be in LABEL_PRINTED state
  (run POST /api/v1/parcels/1/log-print first)

Body:
{
  "qrCode": "PDS-A1B2C3",
  "awbNumber": "AWB-DTDC-001"
}

Assertions:
  - Status: 200
  - Body: { success: true, data: { parcelId: "PDS-A1B2C3", trackingNo: "AWB-DTDC-001", status: "AWB_LINKED" } }
  - OPERATOR scan → status = AWB_LINKED (not auto-dispatched)
  - dispatchDate should be null

Post-Request Verification:
  - GET /api/v1/parcels/1/timeline should show AWB_LINK event
  - Event: { actionType: "AWB_LINK", awbNumber: "AWB-DTDC-001", previousStatus: "LABEL_PRINTED", newStatus: "AWB_LINKED" }

--- TEST 2: Scan + Link AWB (COURIER — Auto-Dispatch) ---
Method: POST
URL: /api/v1/parcels/scan
Headers:
  Authorization: Bearer <COURIER_TOKEN>
  Content-Type: application/json

Pre-condition: Parcel with parcelId "PDS-D4E5F6" must be in LABEL_PRINTED state

Body:
{
  "qrCode": "PDS-D4E5F6",
  "awbNumber": "AWB-BLUEDART-002"
}

Assertions:
  - Status: 200
  - Body: { success: true, data: { status: "DISPATCHED", dispatchDate: "<timestamp>" } }
  - COURIER scan → status jumps directly to DISPATCHED
  - dispatchDate should be populated
