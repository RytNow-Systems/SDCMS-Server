# PUT /orders/:id — Full Graph Update API Testing

## Case 1: Happy Path (Update sender + Add/Remove/Update receivers)
**Method:** PUT
**URL:** /api/v1/orders/1
**Headers:**
- Authorization: Bearer <token>
- Content-Type: application/json
**Body:**
```json
{
  "senderId": 2,
  "senderAddressId": 2,
  "receivers": [
    {
      "receiverDetailsId": 1, 
      "receiverId": 1,
      "receiverAddressId": 1,
      "products": [
        {
          "orderItemId": 1, 
          "variationId": 1,
          "quantity": 5
        },
        {
          "variationId": 2, 
          "quantity": 10
        }
      ]
    },
    {
      "receiverId": 2,
      "receiverAddressId": 2,
      "products": [
        {
          "variationId": 1,
          "quantity": 1
        }
      ]
    }
  ]
}
```
**Expected Outcome:**
- Sender is updated to ID 2.
- Receiver Details ID 1 is updated. Order Item 1 is updated (qty=5). New item (variationId=2) is added. Any other items for Receiver 1 are soft-deleted.
- Receiver Details ID 2 (if it existed) is removed, its items soft-deleted, and its parcel cancelled.
- A new Receiver (party ID 2) is created with 1 product, and a new parcel is generated.
- Response should be a 200 OK with the full updated aggregate graph.

## Case 2: Threshold Block (Order with physical execution)
**Method:** PUT
**URL:** /api/v1/orders/[order_id_with_awb_linked_parcel]
**Headers:**
- Authorization: Bearer <token>
- Content-Type: application/json
**Body:**
```json
{
  "senderId": 1
}
```
**Expected Outcome:**
- 400 Bad Request
- Error: "Cannot update order: physical execution has begun on one or more parcels."

## Case 4: Missing Required Receiver Fields
**Method:** PUT
**URL:** /api/v1/orders/1
**Headers:**

- Content-Type: application/json
**Body:**
```json
{
  "receivers": [
    {
      "receiverDetailsId": 1
    }
  ]
}
```
**Expected Outcome:**
- 400 Bad Request
